# Hormuz Shock Tracker

**An open, reproducible view of the 2026 Iran–US war through the lens of global connectedness.**

Live site: **[pmxlr8.github.io/DS_R_Geopolitics](https://pmxlr8.github.io/DS_R_Geopolitics)** (rebuilt daily)

---

## What this is

On 28 February 2026, a joint US–Israel strike on Iranian nuclear and missile infrastructure triggered the closure of the Strait of Hormuz and the fastest oil-price spike since the 1970s. Most analysis of this shock lives in paywalled consultancy decks. This repository is the **open-source counterpart** — a daily-refreshing Quarto site that ingests World Bank and FRED data into a DuckDB warehouse and re-scores a simplified **DHL Global Connectedness** index around the shock.

The framing borrows openly from the DHL Initiative on Globalization at NYU Stern. The methodology is intentionally transparent enough to fit on one page — see [`pages/methodology.qmd`](pages/methodology.qmd).

## Architecture

```
ingest/*.R           ->  data/raw/*.csv   ->  data/shock.duckdb   ->  Quarto site
  WDI, FRED, events                          (DBI + duckdb)          (ggplot2 + plotly)
                             ^                                              |
                             |                                              v
                 daily GitHub Actions cron                          GitHub Pages
```

- **Ingest.** Live calls to the World Bank API (`WDI`) and FRED's public CSV endpoint, with committed snapshots in `data/raw/` as fallback so the site renders even if an API is down.
- **Warehouse.** A single `data/shock.duckdb` file, regenerated by `build_db.R` on every CI run (so it's gitignored — the snapshots in `data/raw/` are the committed source of truth). DuckDB over SQLite because the workload is analytical: group-bys, window functions, and joins over ~20k rows.
- **Views.** Raw SQL in `sql/` materialises the tables the frontend queries, keeping Quarto chunks terse.
- **Frontend.** One `index.qmd` with four charts + narrative, plus three supporting pages (methodology, data sources, historical analogs).
- **CI.** `.github/workflows/publish.yml` runs daily at 06:00 UTC, refreshes snapshots, rebuilds the warehouse, renders the site, and deploys via GitHub's native Pages pipeline (`actions/deploy-pages`).

## Replicate locally

You need R (≥ 4.2), [Quarto](https://quarto.org/docs/download/), and a working C/C++ toolchain (DuckDB compiles on first install). Then:

```bash
git clone https://github.com/pmxlr8/DS_R_Geopolitics
cd DS_R_Geopolitics
Rscript setup.R        # installs packages
Rscript build_db.R     # rebuilds data/shock.duckdb
quarto preview         # serves the site locally
```

`Rscript setup.R` is idempotent; re-running it is cheap.

## Why these technical choices

- **R** — the language of the target team (NYU Stern DHL Initiative) and the cleanest toolchain for the economist-friendly packages (`WDI`, `imfr`, `comtradr`) that this kind of work depends on.
- **DuckDB** — 10–100× faster than SQLite on analytical queries, reads Parquet natively, and the warehouse is a single file that commits cleanly.
- **Quarto over R Markdown** — first-class TypeScript-quality HTML, built-in navigation, and a pipeline designed around `quarto render` as a CI primitive.
- **GitHub Actions over a scheduler** — no extra infrastructure; cron, build, and deploy in one workflow file.
- **`ggplot2` → `plotly`** — write grammar-of-graphics once, get an interactive plot free. No duplicate code paths.

## Scope and honesty

This is a **measurement** project, not a causal one. It does not claim the war caused any particular move in the charts; it shows what transparent public indicators did over a window in which the war was the dominant news event. The trade-depth score is a simplification of the real DHL Global Connectedness Index, which also spans capital, information, and people flows.
